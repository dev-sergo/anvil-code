## What worked

- ✅ **Detection logic correct.** `.prettierrc.json` in baseline → Prettier fired. Sandbox WITHOUT prettier config (used for [v1.32-c bench](2026-05-04-v1.32-c-task-agents.md)) → Prettier silently skipped, 8 commits across 11 runs without crash.
- ✅ **Local-only `node_modules/.bin/prettier` resolution.** No npx fallback attempts in log. Sandbox `npm install --save-dev prettier` was sufficient.
- ✅ **Cosmetics-only — никогда не блокирует commit.** Even on r1 commit_skipped (validation failed), Prettier was never the cause; commit-skip was upstream (validation).
- ✅ **No wall-time penalty.** Prettier durationMs ≈ 30ms (per log) — invisible in 60-120s task walls.
- ✅ **Diff parity with non-prettier baseline.** Byte-perfect canonical fixes either way — model emits prettier-canonical code naturally on these tasks. **Validates the design**: Prettier is insurance for cases where model emits noise; on already-clean output it's a noop.

## What broke

- ⚠ **Cannot empirically measure noise reduction.** v1.32-a.6 was motivated by [v1.32-a.4 finding](2026-04-30-v1.32-a.4-l4.1-robustness.md) "4/5 runs landed with cosmetic style noise (indent, blank lines, trailing commas)". On this 4-run sample, model emitted byte-perfect canonical code regardless of Prettier — so we can't directly observe "noisy → clean" transformation. The integration is verified end-to-end; the noise-reduction claim rests on the implicit "if model produced noise, Prettier would clean it" — testable only on a larger sample where noise occurs naturally, or with a different / older model that's noisier.
- ⚠ **L4.1 #1 commit_skipped is a v1.32-c done-after-error issue, not v1.32-a.6 issue.** Documented separately in v1.32-c run-file. v1.32-a.6 took no part in that failure (Prettier never reached invocation point).

## Lessons / next iteration ideas

1. **The wiring is correct, but Prettier's value-prop is contingent on model noise.** On qwen-coder-long emitting canonical code, Prettier is a no-op ~100% of the time. Its value emerges if (a) we add noisier models in fallback (e.g. for cost), or (b) we move to 7B-class small-model coder where noise is more common.
2. **`.prettierrc.json` baseline should be the bench default** when measuring diff-shape variance going forward — currently sandbox is bare. Adding prettier devdep + config to permanent sandbox main would mean every L1.1/L4.1 run goes through the prettier post-step. Tradeoff: fewer "raw" measurements of model output style. Decision deferred.
3. **No wall-time regression** is the biggest practical win — operator can leave Prettier wired permanently in production with zero perf cost.

## Decision

v1.32-a.6 wiring **verified empirically**. Documentation gap in CHANGELOG entry now closed via this run-file. Behavior matches design intent: detect → run → log → commit, no failure paths exercised.

**No follow-up needed for v1.32-a.6.** The micro-iteration ships as designed; production benefits start as soon as a target project has prettier configured.
