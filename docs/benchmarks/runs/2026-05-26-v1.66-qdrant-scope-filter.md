## What worked

- Qdrant `packageName` exact-match filter реализован и покрыт unit-тестами
- Hono hitscore стабильный: 5/6 — H4 (requestId middleware) единственное падение
- H5 (getHeader) и H6 (buildUrl) — новые задачи, оба прошли

## What broke

- trpc −1 vs baseline: T5 (maxBodySize) — Tester schema error + Fixer MAX_TOOL_CALLS
- trpc −1 vs baseline: T6 (dataLoader retry) — Reviewer rejected × 3, логика batch concurrency
- H4 (requestId middleware) — Tester runtime fail + Fixer MAX_TOOL_CALLS × 3
- Scope filter не верифицирован в логах (debug уровень, LOG_LEVEL=info)

## Lessons / next iteration ideas

- Поднять `logger.debug → logger.info` для `Qdrant scope filter applied` → видимость в бенчах
- T5 (maxBodySize) и T6 (dataLoader retry) — системные задачи с высокой TypeScript сложностью; рассмотреть увеличение MAX_TOOL_CALLS для Fixer
- H4 (requestId middleware) — TesterAgent runtime fail из-за объекта ReadableStream в snapshot; нужен guard в TesterAgent против non-serializable output
- Регрессия −1 скорее всего model variance, не регрессия алгоритма
