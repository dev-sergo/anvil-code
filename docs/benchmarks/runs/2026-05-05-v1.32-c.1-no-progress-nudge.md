## What worked

- Nudge intercepted premature `done()` in exactly the failure pattern observed in v1.32-c bench (structural-tool error → bail).
- After one nudge, model tried alternative approach and committed in all 4 cases where it fired.
- L3.1 regression-free — `max_tokens: 4096` has no adverse effect on healthy Coder/Fixer runs.

## What broke / limitations

- AC6 formally missed (+34% wall) due to qwen3 thinking-mode overhead added by `max_tokens: 4096` fix. This is infra drift, not a code regression. On healthy runs (no nudge), nudge adds 0 overhead.
- qwen3 as Planner (thinking model) is slower than ideal — ~60s per plan. Consider `LLM_SMALL_MODEL=qwen2` or similar non-thinking model for faster planning.

## Lessons / next iteration ideas

- `max_tokens: 4096` in llamaswap-client is a necessary defensive fix for any thinking-mode model (qwen3, future models). Consider making it configurable via `GenerateOptions.maxTokens`.
- Planner slowness with qwen3 suggests a follow-up: benchmark non-thinking small model alternatives (v1.32-d.4 candidate).
