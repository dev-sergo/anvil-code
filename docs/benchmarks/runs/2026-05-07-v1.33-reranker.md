## What worked

- AC4 (L1.2 + L1.3): обе задачи впервые получили baseline — 2/3 и 3/3 соответственно. L1.3 особенно стабилен (96–112s, 3/3).
- AC5: baseline precision@5 зафиксирован для L2.1 и L2.2 на 91-файловой кодовой базе.
- Reranker работает (порядок файлов изменился vs baseline) и не деградирует результаты.
- BGE-reranker инфраструктура: llama-swap alias `reranker` стабилен, `LlamaSwapClient.rerank()` работает.
- L1.2 r1 и r2: чистая Zod-реализация с safeParse и правильными constraints.

## What broke

1. **L4.1 регрессия (1/3, было 3/3):** Fixer в r1 и r3 создаёт `src/__tests__/user-service.test.ts` вместо фикса `user-service.ts`. Commit_skipped из-за продолжающихся test-failures. Тот же код (r2) работал корректно. Нестабильность Fixer пути.

2. **Precision@5 = 0% для L2.1/L2.2 на target:** server.ts и routes-файлы не входят в top-30 HNSW кандидатов. Причина: задачи описывают что ДОБАВИТЬ (request-logging middleware, Zod schema extraction), а target-кодовая база это не содержит → семантически далеко. Reranker не помогает если файл не в top-30.

3. **Backup-файлы в индексе:** `data/backups/f42a63c1-1777405917096.ts` попадает в retrieval при каждом запросе на target. Нужно добавить `data/backups/**` в .gitignore-style exclude list indexer'а.

4. **Все L2.x задачи failed:** qwen3 Planner на сложной 91-файловой кодовой базе стабильно выдаёт invalid JSON. Проблема не в retrieval, а в Planner capacity на незнакомой codebase.

5. **L1.1 r1 empty commitHash:** nudge → `create` action → changedLines:2 → git commit возвращает пустой hash. Нужно расследовать (возможно prettier нивелирует diff).

## Lessons / next iteration ideas

1. **Exclude backup dirs from indexing:** `data/backups/**` должен быть в exclude-list GraphRetriever.indexCodebase. Это чистый шум в retrieval.

2. **L4.1 Fixer path нестабилен:** Fixer создаёт test-файлы вместо фикса. Нужно добавить scope policy: Fixer не должен создавать тест-файлы если они не были в original writtenFiles. Или добавить read-grants-write scope для user-service.ts.

3. **Precision@5 на target требует BM25:** Для задач типа "add to src/server.ts" dense-only поиск не находит server.ts (файл не описывает что в него добавить). BM25 по symbol names + path components даст точное совпадение "server" → server.ts. → **v1.34 Hybrid search** — именно эта проблема.

4. **L2.x tasks on wrong codebase:** задачи L2.1/L2.2 написаны для sandbox (src/routes/users.ts, src/server.ts) — в target этих путей нет. Нужно либо создать L2.x-target variants, либо использовать другой target (реальный Fastify проект с >50 файлами).

5. **AC7 wall-time:** добавить `logger.info` для rerank latency в LlamaSwapClient.rerank() перед следующим бенчем.

6. **qwen3 parse failures на target:** рассмотреть max_tokens для Planner (аналогично v1.32-c.1 фиксу) или промпт-нудж для valid JSON.
