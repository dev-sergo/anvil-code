# Design — v1.32-d llama-swap backend swap

> **Status:** DRAFT — pre-impl. **Date:** 2026-05-02 (revised after [llama-api-reference.md](../llama-api-reference.md)).
> **Order:** идёт **до** v1.32-c sub-agents — backend меняем на финальный, потом тестируем sub-agents в проде.
> **Link:** [v1.32-c sub-agents design](./v1.32-c-sub-agents.md) (после)

## TL;DR

Заменить `OllamaClient` на `LlamaSwapClient` (OpenAI-compatible API против llama-swap прокси) в [packages/model-router/](../../packages/model-router/). Single endpoint `http://localhost:8080`, **model aliases** (`coder`/`qwen`/`qwen3`/...) маршрутят per-role — llama-swap сам swap'ает модели в VRAM. Ollama-path сохраняется как fallback (env). **Без grammar-constrained tool calls** — отдельная итерация v1.33.

## 1. Goals & non-goals

| | |
|---|---|
| **G1** | `LlamaSwapClient` имплементит то же contract'ом что `OllamaClient` (5 методов: `chat`/`chatStream`/`chatWithTools`/`embed`/`healthCheck`) |
| **G2** | `inline tool-calls recovery` ([ollama-client.ts:273-304](../../packages/model-router/src/ollama-client.ts#L273-L304)) переносится — qwen2.5-coder продолжает работать |
| **G3** | Per-role MODEL ALIAS routing (single URL, multiple models через llama-swap auto-swap) |
| **G4** | Backwards-compat: `LLM_BACKEND=ollama` возвращает старое поведение |
| **G5** | Все 392 unit-теста + новые ≥12 — зелёные. Bench L1.1+L4.1 — wall-time ≤ Ollama, commit-rate identical |
| **NG1** | НЕ trogаем grammar-constrained generation (v1.33) |
| **NG2** | НЕ переписываем agents/* — только `model-router` |
| **NG3** | НЕ управляем llama-swap процессом (operator уже держит как systemd unit) |
| **NG4** | НЕ настраиваем llama-swap config.yaml — модели и aliases — operator's domain |

## 2. Архитектура

```
ModelBackend (interface)               ← новое
    ├── OllamaClient (existing)        ← keep as fallback
    └── LlamaSwapClient (new)          ← default

ModelRouter
    ├── selectBackend()                ← новое: backend instance из env
    ├── selectModel(role, taskMode)    ← возвращает model alias ("coder"/"qwen"/"qwen3")
    └── route* methods                 ← unchanged from outside
```

`ModelBackend` interface:

```ts
export interface ModelBackend {
  chat(messages: AgentMessage[], model: string, options?: GenerateOptions): Promise<string>;
  chatStream(messages: AgentMessage[], model: string, options?: GenerateOptions): AsyncIterable<string>;
  chatWithTools(messages: ToolLoopMessage[], tools: ToolDefinition[], model: string, options?: GenerateOptions): Promise<ToolCallResponse>;
  embed(text: string, model?: string): Promise<number[]>;
  healthCheck(): Promise<boolean>;
}
```

## 3. Ollama → llama-swap API маппинг

| Аспект | Ollama `/api/chat` | llama-swap `/v1/chat/completions` |
|---|---|---|
| Endpoint | `/api/chat` | `/v1/chat/completions` (OpenAI-compat) |
| Stream format | NDJSON `{message:{content}}` | SSE `data: {choices:[{delta:{content}}]}` + `data: [DONE]` |
| Tool calls field | `message.tool_calls` | `choices[0].message.tool_calls` (OpenAI shape) |
| JSON mode | `format:"json"` | `response_format:{type:"json_object"}` |
| Temperature | `options.temperature` | top-level `temperature` |
| Embed endpoint | `/api/embeddings` `{model, prompt}` → `{embedding}` | `/v1/embeddings` `{model, input}` → `{data:[{embedding}]}` (нужно verify в config.yaml) |
| Health | `/api/tags` | `/health` |
| Inline tool-calls fallback | yes (ollama-client.ts:209-215) | **переносим как есть** — qwen models пишут JSON в content и здесь |
| **Уникальное у llama-swap** | — | `/running`, `/api/metrics`, `/v1/models`, `/api/events` SSE |

## 4. llama-swap integration (single URL + model aliases)

llama-swap уже работает у operator'а как remote systemd unit на `<llama-swap-host>:8080`. Сам swap'ает модели в VRAM по `model` field в request body. **Один URL на всё** — наша задача только roles → aliases.

**Per-role alias mapping (default):**

| Role | Alias | Реальная модель | Контекст |
|---|---|---|---|
| coder / fixer / architect / tester | `coder` | qwen2.5-coder-32b | 8K |
| coder для refactor / scale | `qwen-coder-long` | qwen2.5-coder-32b | 16K |
| planner / reviewer | `qwen3` (MoE 3B active) | qwen3.6-35B-A3B | 16K, fastest |
| embed | `embed` | nomic-embed-text-v1.5 | 8K, **768 dim**, batch supported |

**Env config:**

```
LLM_BACKEND=llamacpp                             # default после v1.32-d lands
LLM_URL=http://localhost:8080                  # совместимо с operator's existing LLM env
LLM_LARGE_MODEL=coder                            # alias из llama-swap config.yaml
LLM_LARGE_LONG_MODEL=qwen-coder-long             # для long-context tasks
LLM_SMALL_MODEL=qwen3                            # MoE — fastest для small role
LLM_EMBED_MODEL=embed                            # nomic-embed-text-v1.5, 768 dim
```

`model` поле в request — это alias из llama-swap `config.yaml`. llama-swap auto-load'ит модель в VRAM (cold-start ~5-15s на первом hit; warm — instant).

**Observability bonus** — `/running`, `/api/metrics`, `/api/events` SSE: можем добавить `LlamaSwapClient.metrics()` helper для health dashboard в Phase B-extra (опц.).

## 5. Tool-calling: parity с Ollama

llama-swap (через llama-server) поддерживает `tools` param — verified в [llama-api-reference.md:104-123](../llama-api-reference.md). Пример из доки на `coder` (qwen2.5-coder-32b) работает.

**Inline tool-calls fallback ([extractInlineToolCalls](../../packages/model-router/src/ollama-client.ts#L273)) переносится 1:1.** qwen-models часто emit tool calls как JSON в `content`, не в structured field — это property модели, не backend'а.

**Sampling params на llama-swap богаче чем на Ollama:** `top_k`, `min_p`, `repeat_penalty`, `seed`, `stop`, `presence_penalty`, `frequency_penalty` — проброс через `GenerateOptions` тривиальный, может улучшить detereminizm в bench'ах (`seed=42`).

## 6. Embedding strategy — РЕШЕНО: full llama-swap stack

Operator добавил `embed` alias = `nomic-embed-text-v1-5-f16-ctx-8k`. **768 dim, batch input supported** (verified). Hybrid mode не нужен — Ollama dependency убирается полностью.

### 6.1 Task-prefixes (НОВЫЙ scope)

nomic-v1.5 был обучен с task-префиксами; без них similarity деградирует на 5-10%:

| Контекст вызова embed | Префикс |
|---|---|
| Индексация документов / символов | `search_document: <text>` |
| Поисковый запрос | `search_query: <text>` |

**Wiring:** на rag-system уровне ([packages/rag/src/](../../packages/rag/src/)), **не** в backend client'е — это property модели, не транспорта. Затрагивает оба backend'а (Ollama path тоже выиграет, если останется).

**Конкретные call sites** (под верификацию в Phase B):
- `graph-retriever.ts` `retrieveContextItems(query)` — оборачиваем `query` в `search_query:`
- индексирующий путь (`indexCodebase` → `indexFile` → `embed(symbolText)`) — оборачиваем в `search_document:`

### 6.2 Dimension change → reindex

Текущий index был на Ollama nomic-embed (тоже 768 dim, та же модель — **dimension совпадает**). Но prefixes раньше не использовались → embeddings из старого индекса несовместимы semantically. **Required:** one-shot reindex при первом запуске на v1.32-d. Документируется как `rm -rf data/vectors/* data/graphs/*` + автоматическая reindex на старте.

## 11. Risks

| # | Risk | Severity | Mitigation |
|---|---|---|---|
| R1 | Tool-calling parity между Ollama и llama-swap differs subtly | MED | Parity test (#8 plan); inline-recovery fallback handles edge case |
| R2 | Embedding model swap инвалидирует HNSW index | LOW | Documented reindex step; `data/vectors/` cleanup командой |
| R3 | **llama-swap model-swap latency** на первом запросе после idle (cold-load 5-15s) | MED | Optional warm-up: `LlamaSwapClient.warmup([alias])` на старте API server. Параллельные tasks bypass'ят cold-start если model уже в VRAM |
| R4 | **VRAM constraint** — llama-swap может unload активной модели если concurrent роли требуют другую | MED | `/running` endpoint мониторим; если concurrency > 1 модель → consider keeping one model для всех ролей в v1.32-d, optimize в v1.32-d.1 |
| R5 | SSE parser corner cases (split chunks across `data:` boundary, `[DONE]` terminator) | MED | Тесты на split chunks; reuse NDJSON pattern but adapted to SSE |
| R6 | Inline tool-call recovery less needed на llama-swap (native support лучше) → dead code | LOW | Acceptable; убирается отдельной cleanup итерацией если bench покажет 0 inline-recovered calls |
| R7 | Network latency remote-host vs localhost (Ollama был localhost у operator'а?) | LOW-MED | Verify в Phase A: `curl -w '%{time_total}\n'`. Если remote add'ит >100ms на round-trip, документируется |
