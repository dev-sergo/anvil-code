# Design: v1.69 — Repo Memory v2 (cross-project patterns)

## Problem

`repo_patterns` v1 has three gaps:

1. **No content dedup.** `id = ${taskId}-${attempt}-${Date.now()}` is always unique, so `INSERT OR IGNORE` never fires. The same error ("Cannot find module './X'" without .js) can be stored 20 times; the `LIMIT 5` just cuts by recency, not relevance.

2. **No frequency signal.** Agents see the 5 most *recent* patterns, not the 5 most *common*. A one-off noise pattern ranks equal to one seen in every task.

3. **Per-project isolation.** Each repo has its own `memory.db`. Patterns learned in hono (`writeHead is not a function`) never reach the trpc run, even though both use the same Node HTTP adapter.

## Goals

- Content-dedup: identical issues (after normalization) increment `hit_count`, not add a row.
- Frequency-ranked output: most frequent patterns shown first.
- Cross-project: merge patterns from the current project DB + all other registered project DBs. Label cross-project patterns distinctly in the prompt.

## Non-goals

- Embedding similarity between patterns (too slow locally).
- Automatic pattern expiry / TTL.
- Pattern taxonomy / clustering.

## Schema changes

### `repo_patterns` — new columns (idempotent migration via PRAGMA table_info check)

```sql
-- New columns added at init() if not present:
ALTER TABLE repo_patterns ADD COLUMN project_id  TEXT NOT NULL DEFAULT '';
ALTER TABLE repo_patterns ADD COLUMN hit_count   INTEGER NOT NULL DEFAULT 1;
ALTER TABLE repo_patterns ADD COLUMN issue_hash  TEXT;
ALTER TABLE repo_patterns ADD COLUMN last_seen   TEXT;

-- Dedup index (created once):
CREATE UNIQUE INDEX IF NOT EXISTS repo_patterns_hash_idx
  ON repo_patterns(issue_hash) WHERE issue_hash IS NOT NULL;
```

Existing rows: `issue_hash = NULL`, `hit_count = 1`, `project_id = ''`. The WHERE filter on the UNIQUE index means old NULL rows don't conflict with new rows.

### `normalize(s)` function

```typescript
function normalize(s: string): string {
  return s.toLowerCase().trim().replace(/\s+/g, ' ');
}
function issueHash(issue: string): string {
  return crypto.createHash('sha256').update(normalize(issue)).digest('hex').slice(0, 16);
}
```

## API changes

### `saveRepoPattern` — signature change

```typescript
// Before: saveRepoPattern(id: string, issue: string): void
// After:
saveRepoPattern(projectId: string, issue: string): void
```

Implementation uses `ON CONFLICT(issue_hash) DO UPDATE`:

```sql
INSERT INTO repo_patterns (id, issue, project_id, hit_count, issue_hash, last_seen)
VALUES (?, ?, ?, 1, ?, datetime('now'))
ON CONFLICT(issue_hash) DO UPDATE SET
  hit_count = hit_count + 1,
  last_seen  = datetime('now')
```

`id` = `issueHash + '-' + projectId` (stable, dedup-safe).

### `getRepoPatterns` — sort by frequency

```typescript
getRepoPatterns(limit = 5): RepoPatternRecord[]
// SQL: ORDER BY hit_count DESC, last_seen DESC LIMIT ?
```

### New static method: `MemoryStore.getCrossProjectPatterns`

```typescript
static getCrossProjectPatterns(
  currentProjectId: string,
  registryDbPath: string, // data/projects.db
  dataRoot: string,       // data/
  limit?: number,         // default 5
): RepoPatternRecord[]
```

1. Opens `registryDbPath` read-only, reads all `project.id` values.
2. For each `id ≠ currentProjectId`: opens `data/projects/{id}/memory.db` read-only (skips if missing).
3. Queries `SELECT issue, issue_hash, hit_count, project_id FROM repo_patterns WHERE issue_hash IS NOT NULL ORDER BY hit_count DESC LIMIT ?` per sibling DB.
4. Merges all results, deduplicates by `issue_hash` (sums `hit_count`), sorts by `hit_count DESC`.
5. Returns top `limit` records with `isLocal = false`.

## Orchestrator changes

### Constructor: add `projectId` + `dataRoot` fields

```typescript
constructor(
  private projectId: string,    // NEW — passed from project-manager
  private dataRoot: string,     // NEW — config.dataRoot
  ...existingArgs,
)
```

### `runTask` initialization

```typescript
// Before:
this.repoPatterns = this.store.getRepoPatterns().map(p => p.issue);

// After:
const local  = this.store.getRepoPatterns(5);
const cross  = MemoryStore.getCrossProjectPatterns(
  this.projectId, config.projects.registryPath, this.dataRoot, 5
);
this.repoPatterns = mergePatterns(local, cross); // RepoPatternRecord[]
```

`mergePatterns`: dedup by `issue_hash`, local patterns marked `isLocal=true`, cross marked `isLocal=false`. Sort: `hit_count DESC`.

### `saveRepoPattern` call site

```typescript
// Before:
this.store.saveRepoPattern(`${taskId}-${attempt}-${Date.now()}`, issue);

// After:
this.store.saveRepoPattern(this.projectId, issue);
```

### `repoPatterns` field type change

```typescript
// Before:  private repoPatterns: string[] = [];
// After:   private repoPatterns: RepoPatternRecord[] = [];
```

All `buildPromptContext(...)` calls pass `repoPatterns: this.repoPatterns`.

## Acceptance criteria

1. Inserting the same issue twice → one row with `hit_count = 2`, not two rows.
2. `getRepoPatterns()` returns patterns sorted by `hit_count DESC`.
3. Cross-project patterns loaded from all registered project DBs (≠ current project).
4. Prompt section shows `[×N]` prefix and `(cross-project)` label where applicable.
5. `pnpm build` and `pnpm test` green (all packages).
6. No bench regression: system starts, tasks complete, patterns are saved and loaded correctly.
