### Task Matrix (12 tasks × 2 targets = 24 runs)

#### L5-A: Additive Complex (4-5 файлов, новые абстракции)

| # | Task | Files | Expected |
|---|---|---|---|
| A1 | Add full user pagination: GET /users returns {data, total, page, limit, totalPages}. UserService.list() accepts {page, limit}. Add PaginatedResponse<T> type. | 3 | ✅ |
| A2 | Add user tags: Tag interface (id, name, color), TagService with CRUD, add tags[] to User, GET /users/:id/tags, POST /users/:id/tags/:tagId | 4-5 | ⚠️ |
| A3 | Add request/response logging: middleware logs {method, url, statusCode, durationMs, requestId} as JSON. Separate log file output. | 3 | ✅ |
| A4 | Add health check with dependencies: GET /health returns {status, db: {connected, latency}, uptime, version}. Simulate DB check with Map size. | 2 | ✅ |

#### L5-B: Structural Refactoring (не потерять поведение)

| # | Task | Files | Expected |
|---|---|---|---|
| B1 | Extract route handlers into controller classes: UsersController with static methods list(), get(), create(). Routes file uses controller. Same HTTP behavior. | 3 | ⚠️ |
| B2 | Add repository pattern WITHOUT breaking tests: UserRepository interface, InMemoryUserRepository, but keep UserService.create() etc as static methods delegating to singleton repo. | 3 | ⚠️ |
| B3 | Migrate from in-memory Map to a class-based Store<T> with get/set/delete/values/has methods. Refactor UserService to use Store<User>. | 2 | ✅? |

#### L5-C: Algorithmic (нетривиальная логика)

| # | Task | Files | Expected |
|---|---|---|---|
| C1 | Add LRU cache to UserService.get(): cache last 50 lookups, evict oldest on overflow. Track hits/misses, expose via GET /users/cache-stats. | 2 | ⚠️ |
| C2 | Add debounced search index: rebuild a sorted search index on writes, binary search on reads. GET /users?search= uses index. | 2 | ⚠️ |
| C3 | Add TTL-based session tokens: POST /users/:id/token returns {token, expiresAt}. GET /users/me with Authorization: Bearer validates token. Tokens expire after 1h. | 3 | ⚠️ |

#### L5-D: Maximum Complexity

| # | Task | Files | Expected |
|---|---|---|---|
| D1 | Add WebSocket endpoint GET /users/live that streams user.created events to all connected clients. UserService emits events, server fans out. | 3-4 | ❌? |
| D2 | Full CQRS split: separate UserCommandService (create/update/delete, emits events) and UserQueryService (read-only, caches). Routes delegate to both. | 4-5 | ❌? |

### Success Criteria

| Tier | Criterion |
|---|---|
| A tasks | ≥3/4 correct commits on sandbox; ≥2/4 on target |
| B tasks | ≥2/3 correct commits (structural is hard) |
| C tasks | ≥1/3 correct commits |
| D tasks | ≥0/2 (these are stretch goals) |
| **Overall** | ≥6/12 on sandbox, ≥4/12 on target |

### Scoring rubric per task

- ✅ **Full credit**: commit + correct logic + no runtime bugs
- ⚠️ **Partial**: commit + correct TypeScript + minor logic bug
- ❌ **Fail**: no commit OR broken TypeScript
