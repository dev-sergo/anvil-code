# v1.67 — SQLite symbol table + multi-hop queries

## TL;DR

Переносим граф символов из in-memory `Map + reverseIndex` + `graph.json` в SQLite таблицы `symbols` / `dependencies`.  
Заменяем BFS-обход `getTransitiveCallers()` на recursive CTE — получаем depth 3 без O(n²) и без cap по `callersPerSymbol`.  
Scope filter log поднимаем с `debug` → `info` (закрываем долг v1.66).

---

## Проблема

| Симптом | Корень |
|---|---|
| `graphHops=1` по умолчанию (cap `callersPerSymbol=3`) | Dense граф (trpc 3938 символов) даёт noise при depth>1 в BFS |
| `getTransitiveCallers` не видит cross-service цепочки глубже 1 | Reverse index: O(N×callersPerSymbol) без индексов |
| Scope filter не видно в логах | `logger.debug` при `LOG_LEVEL=info` |
| CodeGraph перестраивает reverseIndex при каждой загрузке | JSON → Map → BFS rebuild — ~50–200ms на 3938 символах |

---

## Цель

- Таблицы `symbols` + `dependencies` в `memory.db` (уже есть `better-sqlite3`)
- `SymbolTable` класс: CRUD + `getTransitiveCallers(seeds, depth)` через recursive CTE
- `GraphRetriever.indexFile()` пишет в SQLite параллельно CodeGraph (JSON остаётся как fallback)
- `GraphRetriever.retrieveContextItems()` использует SQL для caller expansion вместо BFS
- Миграция: `graph.json` → SQLite (idempotent one-shot скрипт)
- `RAG_GRAPH_HOPS` default: 1 → 3 (SQL indexed, безопасно увеличить)

## Схема

```sql
-- Добавляется в MemoryStore.init()

CREATE TABLE IF NOT EXISTS symbols (
  id      INTEGER PRIMARY KEY AUTOINCREMENT,
  name    TEXT NOT NULL,
  kind    TEXT NOT NULL,
  file_path TEXT NOT NULL,
  start_line INTEGER NOT NULL,
  end_line   INTEGER NOT NULL,
  body    TEXT,
  package_name TEXT,
  UNIQUE(name, file_path)
);
CREATE INDEX IF NOT EXISTS idx_symbols_name ON symbols(name);
CREATE INDEX IF NOT EXISTS idx_symbols_file ON symbols(file_path);

CREATE TABLE IF NOT EXISTS dependencies (
  from_id INTEGER NOT NULL REFERENCES symbols(id) ON DELETE CASCADE,
  to_id   INTEGER NOT NULL REFERENCES symbols(id) ON DELETE CASCADE,
  PRIMARY KEY (from_id, to_id)
);
CREATE INDEX IF NOT EXISTS idx_deps_to ON dependencies(to_id);
```

`dependencies.to_id` — индекс нужен для "кто вызывает X?" в CTE.  
`UNIQUE(name, file_path)` — `INSERT OR IGNORE` делает upsert idempotent.  
Нет `kind`/`weight` в dependencies — упрощение для v1.67; regex не даёт надёжный kind; добавим в v1.68 с AST.

## Recursive CTE

```sql
WITH RECURSIVE callers(id, depth) AS (
  SELECT from_id, 1
  FROM   dependencies
  WHERE  to_id IN (SELECT id FROM symbols WHERE name IN (?seeds))

  UNION ALL

  SELECT d.from_id, c.depth + 1
  FROM   dependencies d
  JOIN   callers c ON d.to_id = c.id
  WHERE  c.depth < ?maxDepth
)
SELECT DISTINCT s.name, s.kind, s.file_path, s.start_line, s.end_line, s.body, c.depth
FROM   callers c
JOIN   symbols s ON s.id = c.id
ORDER  BY c.depth, s.name
LIMIT  ?limit;
```

`UNION ALL` без `NOT EXISTS` guard — SQLite CTE по умолчанию не проверяет cycles; добавим dedupe через `GROUP BY id` в outer `SELECT DISTINCT`.

## Архитектурные фазы

### A — Schema + SymbolTable класс
- `MemoryStore.init()`: добавить DDL для `symbols` + `dependencies`
- Новый файл `packages/rag/src/symbol-table.ts`:
  - `upsertFile(filePath, symbols[])` — вставка/замена всех символов файла + edges
  - `removeFile(filePath)` — `DELETE FROM symbols WHERE file_path = ?` (CASCADE удаляет edges)
  - `getTransitiveCallers(seeds: string[], maxDepth: number, limit?: number): SymbolRow[]`

### B — Интеграция в GraphRetriever
- `indexFile()`: после `codeGraph.addFile()` вызвать `symbolTable?.upsertFile(filePath, symbols)`
- `retrieveContextItems()`: заменить `codeGraph.getTransitiveCallers(...)` на `symbolTable?.getTransitiveCallers(...)`, с fallback на CodeGraph BFS если SQLite недоступен
- `flush()`: ничего менять не нужно (SQLite writes синхронные)
- `loadFromDisk()`: добавить `symbolTable` в конструктор (через `store as SymbolTableStore`)

### C — Config
- `RAG_GRAPH_HOPS` default: 1 → 3
- `RAG_CALLERS_PER_SYMBOL` оставить (используется как SQL LIMIT)

### D — Migration script
```
packages/rag/src/scripts/migrate-graph-to-sqlite.ts
```
- Загружает `graph.json` (если есть)
- `INSERT OR IGNORE` каждый символ → получает rowid
- Для каждого символа: `extractDependencies(sym)` → резолвит имена → вставляет edges
- Идемпотентен: можно перезапустить

### E — Log level fix
- `graph-retriever.ts:253`: `logger.debug` → `logger.info` для "Qdrant scope filter applied"

## Риски

| Риск | Вероятность | Митигация |
|---|---|---|
| Cycles в dependency graph → infinite CTE | LOW | `UNION ALL` + `DISTINCT` + `LIMIT 200` гарантируют завершение |
| SQLite FOREIGN KEY по умолчанию OFF | MEDIUM | `PRAGMA foreign_keys = ON` в `MemoryStore.init()` |
| `extractDependencies` даёт много ложных edges (regex) | HIGH | Не блокирует — шум в edges не хуже текущего BFS |
| graph.json отсутствует (свежая установка) | LOW | Migration script: ранний выход, символы запишутся при первом indexCodebase |
